With Proxmox running and Pi-hole handling DNS, the infrastructure was stable. Now it was time to use it for what the homelab was actually built for: a safe, isolated environment to practice offensive security. This post covers standing up the attacker and target machines and running the first reconnaissance against the lab.

---

## Lab Design

The setup is two VMs on the same internal Proxmox network, isolated from the rest of my LAN:

- **Attacker:** Kali Linux — purpose-built for offensive security, loaded with tools out of the box.
- **Target:** Ubuntu Server running **DVWA** (Damn Vulnerable Web Application) — a deliberately insecure web app designed for exactly this kind of practice.

Keeping both VMs inside Proxmox's virtual network means all attack traffic stays contained. Nothing reaches my real LAN unless I explicitly configure it to.

## Reconnaissance

Before touching DVWA, I ran some recon from the Kali VM to get familiar with the network and confirm what was reachable. Standard practice — map the environment before you start poking at it.

### Connectivity Check

A basic ping first to confirm Kali could reach Pi-hole on the network.

![Ping to Pi-hole IP](/assets/img/posts/2025-09-11-homelab-v1-4/ping-pihole-ip.png){: w="700" h="400" }_Ping confirming LAN reachability_

### Nmap Scans

From there I ran a series of Nmap scans against the gateway and Pi-hole container, escalating from host discovery to full service enumeration.

A host discovery scan to map what's on the network:

![Nmap gateway scan](/assets/img/posts/2025-09-11-homelab-v1-4/nmap-gateway-scan.png){: w="700" h="400" }_Nmap host discovery_

A service scan to enumerate open ports and what's running on them:

![Nmap service scan](/assets/img/posts/2025-09-11-homelab-v1-4/nmap-service-scan.png){: w="700" h="400" }_Nmap service enumeration_

An aggressive scan for OS detection, service versions, and default script results:

![Nmap aggressive scan](/assets/img/posts/2025-09-11-homelab-v1-4/nmap-aggressive-scan.png){: w="700" h="400" }_Nmap aggressive scan_

All results — along with a Nikto web vulnerability scan — were saved to log files and committed to the repo. Reading real scan output against machines you built yourself makes the format click in a way documentation doesn't. You know what's supposed to be there, so you actually understand what Nmap is telling you.
